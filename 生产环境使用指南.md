## 快速开始

### 1. 登录系统

访问 `http://127.0.0.1:5000`，使用 admin 账号登录。

### 2. 添加真实项目

有两种方式添加项目：

#### 方式 A：通过 Web 界面

1. 登录后点击「项目」→「新建项目」
2. 填写项目信息：
   - 项目名称
   - Git 仓库 URL（或本地路径）
   - 语言/框架
3. 保存

#### 方式 B：通过 API

```bash
# 获取 JWT token
TOKEN=$(curl -s http://127.0.0.1:5000/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"admin@sentinel.io","password":"admin123"}' \
  | python -c "import sys,json; print(json.load(sys.stdin)['token'])")

# 创建项目
curl -s http://127.0.0.1:5000/api/projects \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "我的真实项目",
    "repo_url": "https://github.com/your-org/your-repo",
    "language": "Python",
    "framework": "Django"
  }'
```

### 3. 启动白盒扫描

#### 方式 A：通过 Web 界面

1. 进入项目详情页
2. 点击「启动扫描」
3. 选择扫描类型：「白盒扫描（SAST）」
4. 确认启动

#### 方式 B：通过 API

```bash
# 启动扫描（假设项目 ID 为 1）
curl -s -X POST http://127.0.0.1:5000/api/scans \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "project_id": 1,
    "scan_type": "sast",
    "branch": "main"
  }'
```

### 4. 查看扫描结果

- 扫描完成后，在「漏洞」页面查看发现的安全问题
- 每个漏洞包含：
  - 漏洞类型（OWASP Top 10 / CWE）
  - 严重等级（Critical / High / Medium / Low）
  - 文件路径 + 行号
  - 修复建议

## 增强 SAST 扫描器

已集成增强版白盒扫描器，覆盖：

| 语言 | 规则数 | 覆盖漏洞类型 |
|------|---------|--------------|
| Python | 46 条 | SQL 注入、命令注入、XSS、SSRF、硬编码密钥、不安全的反序列化、弱加密等 |
| JavaScript/TypeScript | 26 条 | 同上 + 原型链污染、不安全的 JWT 处理等 |

扫描器会自动识别项目语言并应用对应规则。

---

## 扫描真实代码的最佳实践

1. **先扫小项目**：第一次使用建议用一个小型真实项目测试，熟悉流程
2. **检查扫描结果**：增强扫描器可能会报告较多漏洞，需要人工验证
3. **修复优先级**：先修 Critical 和 High 等级漏洞
4. **集成 CI/CD**：扫描器可集成到 CI/CD 流水线，每次提交自动扫描
