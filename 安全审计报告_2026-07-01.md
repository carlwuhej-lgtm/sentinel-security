## 🔴 Critical 严重漏洞（必须立即修复）

### C-01 | JWT 密钥硬编码 — 可伪造任意用户令牌

| 项目 | 详情 |
|------|------|
| **CWE** | CWE-798: 硬编码凭证 |
| **文件** | `backend/config.py:28` |
| **风险** | 攻击者可离线签发任意 JWT Token，冒充管理员 |

```python
# config.py:28 — 默认密钥硬编码
JWT_SECRET = os.environ.get("SENTINEL_JWT_SECRET", "sentinel-jwt-secret-2026")
```

**攻击路径**：
1. 攻击者获取源码或配置文件（公开仓库/内部泄露/逆向）
2. 使用默认密钥 `sentinel-jwt-secret-2026` 伪造 JWT
3. 在 payload 中设置 `role: "admin"`，获得全部管理权限

**修复方案**：
```python
# config.py
import os, secrets
JWT_SECRET = os.environ.get("SENTINEL_JWT_SECRET")
if not JWT_SECRET:
    raise RuntimeError("❌ SENTINEL_JWT_SECRET 环境变量未设置！请生成随机密钥。")
# 或在启动时自动生成并持久化：
# JWT_SECRET = os.environ.get("SENTINEL_JWT_SECRET") or secrets.token_hex(64)
```

**优先级**：🚨 立即修复

### C-02 | Webhook Token 默认放行 — 未认证即可触发扫描

| 项目 | 详情 |
|------|------|
| **CWE** | CWE-306: 缺失关键功能认证 |
| **文件** | `backend/routes/webhooks.py:40-41` |
| **风险** | 任何人可匿名触发 CI/CD 扫描，构造恶意请求 |

```python
# webhooks.py:40-41
if not db_stored:
    return True  # 开发模式：未配置 token 时允许所有请求
```

**攻击路径**：
1. 攻击者向 `POST /api/webhooks/scan?token=anything` 发送请求
2. 由于数据库中 token 为空，鉴权自动通过
3. 可触发任意项目的扫描、消耗资源、注入恶意扫描结果

**修复方案**：
```python
def _verify_token(db) -> bool:
    row = db.execute("SELECT token FROM webhook_config WHERE id=1").fetchone()
    db_stored = row["token"] if row else ""

    # ❌ 删除这段自动放行的逻辑
    # if not db_stored:
    #     return True

    if not db_stored:
        return False  # 未配置 token 时拒绝所有请求

    token = request.args.get("token", "") or request.headers.get("X-Sentinel-Token", "")
    return hmac.compare_digest(token, db_stored)
```

**优先级**：🚨 立即修复

## 🟠 High 高危漏洞

### H-01 | 工具 API Key 明文暴露 — GET /api/tools 泄露全部凭据

| 项目 | 详情 |
|------|------|
| **CWE** | CWE-200: 敏感信息泄露 |
| **文件** | `backend/routes/tools.py:13` |
| **风险** | 任何登录用户可查看所有集成工具的 API Key |

```python
# tools.py:13 — SELECT * 包含了 api_key 字段！
rows = db.execute("SELECT * FROM tools ORDER BY tool_type, name").fetchall()
return jsonify([dict(r) for r in rows])
```

**修复方案**：
```python
# 列表接口移除敏感字段
rows = db.execute(
    "SELECT id, name, tool_type, description, endpoint, enabled, created_at FROM tools"
).fetchall()

# 详情接口保留敏感字段但做脱敏
row = dict(row)
row["api_key"] = "****" + row["api_key"][-4:] if row["api_key"] else ""
```

### H-02 | SQL 注入（ORDER BY 子句）— assets/rules 接口存在注入点

| 项目 | 详情 |
|------|------|
| **CWE** | CWE-89: SQL 注入 |
| **文件** | `backend/routes/assets.py:108`、`rules.py:64` 等 |
| **风险** | 可执行任意 SQL（受 ORDER BY 语法限制但可信息泄露） |

```python
# assets.py:108-110 — sort 参数仅做了非彻底的字符串替换
safe_sort = sort.replace("'", "").replace('"', "").replace(";", "")
q += f" ORDER BY {safe_sort} {safe_order}"
```

**攻击 Payload 示例**：
```
?sort=CASE WHEN (SELECT unicode(substr(password,1,1)) FROM users WHERE id=1)>100 THEN risk_score ELSE name END
```

**修复方案**：
```python
# 白名单校验（最安全）
ALLOWED_SORT_FIELDS = {
    "name", "asset_type", "risk_score", "risk_level", "created_at",
    "updated_at", "status", "last_scan_date", "last_vuln_count"
}
sort = request.args.get("sort", "updated_at")
if sort not in ALLOWED_SORT_FIELDS:
    sort = "updated_at"
```

### H-03 | SMTP 密码明文存储

| 项目 | 详情 |
|------|------|
| **CWE** | CWE-312: 敏感数据明文存储 |
| **文件** | `backend/routes/settings.py:162-163` |
| **风险** | 数据库泄露 = 邮件服务器凭据泄露 |

邮件服务器的密码直接以明文存储于 SQLite 数据库的 `email_config` 表中。虽然 API 返回时做了脱敏（显示 `****`），但数据库文件一旦泄露（如备份文件、SQLite 文件被下载），攻击者可直接读取 SMTP 密码。

**修复方案**：使用对称加密存储（AES-256-GCM），加密密钥从环境变量读取。

---

### H-04 | 无账户级别锁定 — 分布式暴力破解可绕过

| 项目 | 详情 |
|------|------|
| **CWE** | CWE-307: 暴力破解防护不足 |
| **文件** | `backend/routes/security.py` |
| **风险** | 使用 botnet 可绕过 IP 锁定，无限尝试密码 |

当前锁定机制基于 **IP 地址**，而非**用户账户**。攻击者使用多个 IP（僵尸网络/代理池）可以对单个账户发起无限次登录尝试。

**修复方案**：在用户表中增加 `login_fail_count` 和 `locked_until` 字段（数据表已有），登录时同时检查账户级锁定和 IP 级锁定。

## 🟡 Medium 中危漏洞

### M-01 | JWT 无吊销机制 — 令牌泄露后无法撤销

| 文件 | `backend/routes/auth.py:88-97` |
|------|------|
| JWT 签发后在 24 小时内有效，无黑名单/注销机制。令牌一旦泄露（如 XSS、日志泄露），攻击者可在有效期内持续利用。 | |

**修复方案**：实现 Token 黑名单（Redis/数据库），或在 JWT 中包含递增的 token version。

### M-02 | 密码策略过弱 — 仅要求 6 位字符

| 文件 | `backend/routes/auth_routes.py:45` |
|------|------|
| `len(password) < 6` — 没有大写/数字/特殊字符要求，`123456` 是合法密码。 | |

**修复方案**：
```python
if len(password) < 8:
    return jsonify({"error": "密码至少 8 位"}), 400
if not re.search(r'[A-Z]', password):
    return jsonify({"error": "密码需包含大写字母"}), 400
if not re.search(r'[a-z]', password):
    return jsonify({"error": "密码需包含小写字母"}), 400
if not re.search(r'[0-9]', password):
    return jsonify({"error": "密码需包含数字"}), 400
```
