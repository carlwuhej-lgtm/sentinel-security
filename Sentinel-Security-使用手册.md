## 1. 产品概述

**Sentinel Security（哨兵）** 是一款面向企业内部研发团队的 **DevSecOps 应用安全编排平台**。它将应用安全（AppSec）能力从分散的工具整合为统一的工作台，覆盖「资产登记 → 自动化扫描 → 漏洞管理 → 修复工单 → 事件调查 → 合规报告」的完整闭环，并内置 AI 智能分析与安全知识库。

### 1.1 核心能力

| 能力域 | 说明 |
|--------|------|
| **多类型扫描编排** | 支持 SAST（静态应用安全测试）、SCA（软件成分分析）、DAST（动态应用安全测试）、Secret 检测四类扫描，可批量发起、统一跟踪。 |
| **漏洞全生命周期管理** | 按严重级别（Critical/High/Medium/Low）与 SLA 跟踪，支持指派、验证、批量修复、忽略、转工单。 |
| **AI 智能分析** | 对单条漏洞给出根因分析、修复方案、前后代码对比、攻击路径还原与业务影响量化；并提供 AI 安全顾问对话。 |
| **告警与工单闭环** | 扫描产生的风险自动转为告警，可一键创建修复工单并跟踪状态。 |
| **资产与规则管理** | 登记应用资产、自定义检测规则与忽略规则，支撑策略化安全治理。 |
| **合规与报告** | 生成安全总览、漏洞明细、SLA 合规、趋势分析、合规检查清单等报告（JSON/Markdown/CSV/PDF）。 |
| **审计追溯** | 记录平台所有操作与安全风险事件，支持多维度检索与导出，满足合规审计要求。 |
| **CI/CD 门禁** | 通过 Webhook 与定时调度将安全卡点嵌入研发流水线（GitLab / GitHub / Jenkins）。 |

### 1.2 技术架构

- **后端**：Flask 3（Blueprint 模块化，26 个路由模块），SQLite 持久化。
- **前端**：React 18 + TypeScript + Vite + Tailwind CSS，单页应用。
- **认证**：JWT（HS256，含 `token_version` 吊销机制）+ PBKDF2 密码哈希（26 万轮）。
- **部署**：支持 Docker Compose 一键部署，或前后端分离本地运行；内置 Nginx 反向代理。

## 2. 快速开始

### 2.1 访问平台

平台默认通过 Web 访问（部署后通常为 `http://<服务器地址>:5000`）。登录页即平台首页。

### 2.2 默认管理员账号

首次部署后，平台内置一个默认管理员：

```
邮箱：admin@sentinel.io
密码：admin123
```

> ⚠️ **安全提示**：默认密码仅用于首次登录。出于安全合规要求，**首次使用默认密码登录会被强制要求修改密码**；修改成功后才能正常使用平台。请务必设置一个强密码，并妥善保管。

### 2.3 首次登录流程

1. 在登录页输入邮箱与密码，点击「登录」。
2. 若使用的是默认初始密码，系统弹出**强制改密**窗口（不可关闭），设置新密码后继续。
3. 登录成功后进入「今日概览」仪表盘。

### 2.4 账号注册说明（邀请模式）

出于安全考虑，平台**默认关闭公开自助注册**。普通用户无法自行注册账号，需由**管理员在「用户管理」中创建账号**（邀请模式）。管理员在用户管理页填写姓名、邮箱、初始密码、角色与状态即可开通账号。

> 仅当管理员在「设置 → 外观 → 注册策略」中手动开启「开放公开注册」后，登录页才会出现注册入口。生产环境**不建议**开启。

### 2.5 界面布局

登录后界面由以下部分组成：

- **左侧导航栏**：按分组组织全部功能模块（概览 / 工作流 / 分析 / 配置 / 合规 / 系统），并按角色显示可用项。
- **顶部栏**：全局搜索、**命令面板快捷键（Ctrl/Cmd + K）**、语言切换、主题切换、用户菜单（含修改密码、登出）。
- **主内容区**：当前模块的功能视图。

## 3. 角色与权限模型

平台采用基于角色的访问控制（RBAC）。共有 4 种内置角色：

| 角色 | 中文 | 定位 |
|------|------|------|
| `admin` | 安全管理员 | 平台最高权限，含用户管理与系统设置。 |
| `security_analyst` | 安全分析师 | 负责漏洞研判、扫描、规则、报告，但不可管理用户与系统设置。 |
| `developer` | 开发人员 | 只读为主，可更新**被指派给本人**的漏洞（修复状态）。 |
| `viewer` | 只读用户 | 仅可查看，无写权限。 |

### 3.1 权限矩阵

下表为各角色可执行的典型操作（`✓` 表示允许）：

| 资源 / 操作 | admin | security_analyst | developer | viewer |
|-------------|-------|------------------|-----------|--------|
| 项目：查看 | ✓ | ✓ | ✓ | ✓ |
| 项目：新建 / 编辑 / 删除 | ✓ | — | — | — |
| 扫描：发起 / 查看 | ✓ | ✓ | 查看 | 查看 |
| 漏洞：查看 | ✓ | ✓ | ✓ | ✓ |
| 漏洞：更新 / 指派 / 验证 | ✓ | ✓ | 仅本人指派 | — |
| 漏洞：删除 / 批量删除 | ✓ | — | — | — |
| 资产：新建 / 编辑 / 删除 | ✓ | ✓ | — | — |
| 规则：新建 / 编辑 / 删除 | ✓ | ✓ | — | — |
| 报告：生成 / 查看 / 导出 | ✓ | ✓ | 查看 | 查看 |
| 用户：管理（增删改角色/状态） | ✓ | — | — | — |
| 系统设置：修改 | ✓ | — | — | — |
| 知识库：撰写 / 编辑 / 删除 | ✓ | ✓ | — | — |
| 审计日志：查看 | ✓ | ✓ | — | ✓ |

### 3.2 说明

- 界面上部分业务按钮对所有登录用户可见，但**实际是否允许执行取决于服务端 RBAC 校验**。若当前角色无权操作，后端会返回 `403 权限不足`。请以账号实际权限为准。
- **用户管理**入口仅对 `admin` 可见；知识库的撰写/编辑仅 `admin` 与 `security_analyst` 可执行；**设置**中的「Logo 上传」「注册策略」仅 `admin` 可操作。

## 4. 核心概念与 DevSecOps 闭环

理解以下核心概念，有助于高效使用平台：

- **项目（Project）**：被扫描的代码仓库或应用，是扫描的资产来源。
- **资产（Asset）**：从项目同步或手动登记的应用资产（如服务、组件），关联其开放漏洞与风险分。
- **扫描（Scan）**：针对项目发起的安全检测任务，可运行多种工具（SAST/SCA/DAST/Secret）。
- **漏洞（Vulnerability）**：扫描发现的具体安全风险，按严重级别与 SLA 跟踪。
- **告警（Alert）**：扫描或规则触发的风险通知，需确认/关闭。
- **工单（Ticket）**：用于跟踪漏洞或告警修复任务的工作项。
- **规则（Rule）**：自定义检测规则或忽略规则，用于策略化治理。

### 闭环工作流

```
  项目/资产
     │
     ▼
  发起扫描（SAST/SCA/DAST/Secret）
     │
     ▼
  发现漏洞 ───────► 生成告警
     │                  │
     │                  ▼
     │              确认/关闭告警
     ▼
  漏洞管理（指派 / SLA / AI 分析 / 验证）
     │
     ▼
  创建修复工单 ──► 开发人员修复 ──► 验证关闭
     │
     ▼
  事件调查（Investigation）聚合研判
     │
     ▼
  生成报告 / 审计追溯（合规闭环）
```

AI 分析、知识库、设置等模块贯穿上述全流程，提供辅助决策与平台治理能力。

## 5. 功能模块详解

> 导航按以下分组组织：**概览 / 工作流 / 分析 / 配置 / 合规 / 系统**。

### 5.1 概览 — 今日概览

**入口**：左侧导航「今日概览」（首页 `/`）。

仪表盘汇总当前安全态势，支持时间范围切换（`今日 / 近 7 天 / 近 30 天 / 全部`）：

- **统计卡片**：开放漏洞数、待处理告警数、开放工单数、SLA 超时数、新增数、修复率。
- **紧急漏洞清单**：高优先级的待处理漏洞。
- **新告警 / SLA 提醒**：近期告警与即将/已经超时的 SLA 项。
- **快捷操作**：一键发起扫描、新建项目。
- **首次使用引导**：无数据时的空态引导。

> 全员可见。

### 5.2 工作流

#### 5.2.1 漏洞管理（Vulnerabilities）

**核心业务模块**，用于全量漏洞追踪与 SLA 管理。

- **统计条**：总计 / Critical / High / Medium / Low / SLA 超时 / 已修复。
- **筛选与搜索**：按严重级别、SLA 超时、已修复状态筛选；搜索 CVE / 标题 / 文件路径。
- **批量操作**：批量修复、批量删除（勾选多条后操作）。
- **单条操作**：AI 分析、重新验证（reverify）、删除。
- **漏洞详情抽屉**（三个标签）：
  - **详情**：文件路径、发现工具（可跳转工具页）、指派人、发现时间、SLA 状态与到期、描述、工具自带修复建议、重新指派。
  - **AI 修复**：根因分析、修复方案、前后代码对比、验证建议。
  - **AI 分析**：风险评分（ai_risk_score）、攻击路径还原、利用难度、业务影响量化、缓解建议。
- **状态流转**：待处理 → 处理中 → 已修复；支持「忽略」。
- **一键转工单**：从漏洞直接创建修复工单并带入上下文。

#### 5.2.2 扫描中心（Scans）

发起与跟踪各类安全扫描，配置 CI/CD 门禁与定时调度。

- **扫描列表**：按项目筛选、分页；运行中扫描每 3 秒自动刷新；展开查看漏洞详情。
- **新建批量扫描**：多选项目 × 多工具（SAST / SCA / DAST / SECRET）。
- **删除扫描记录**。
- **CI/CD 安全门禁**：查看 GitLab / GitHub / Jenkins 集成代码片段；重新生成 Webhook Token；配置门禁规则（如 Critical 阻断、High 告警、Medium/Low 放行）。
- **定时调度**：设置 cron / 间隔周期，启用/停用、立即执行、编辑、删除。

#### 5.2.3 工单中心（Tickets）

漏洞/告警的修复跟踪闭环。

- 统计标签、搜索、状态筛选；工单列表。
- 详情弹窗：状态切换、来源信息、评论增删。
- 新建/编辑工单：标题、描述、优先级、状态、负责人（从用户列表选择）。
- 删除工单。

#### 5.2.4 告警中心（Alerts）

集中处理扫描产生的告警。

- 统计：今日告警 / 待处理 / 今日新增 / 已关闭；级别分布。
- 筛选：状态 + 严重级别 + 关键词；分页。
- 批量：标记已确认、批量关闭。
- 单条流转：未处理 → 确认 / 关闭。
- 通知渠道：可配置飞书机器人等（见设置）。

### 5.3 分析

#### 5.3.1 事件调查（Investigation）

聚合待处理漏洞与告警进行关联调查，采用三栏布局（事件列表 / 详情 / 关联上下文）。

- 聚合待处理漏洞与告警；按严重/高危/待处理/类型筛选；搜索。
- 详情查看原始漏洞/告警信息。
- **AI 分析**：分析结果自动保存回漏洞的 `ai_analysis` 字段。
- **转工单**、关联同类问题（同文件路径 / 同扫描器）。
- AI 安全顾问对话。

#### 5.3.2 AI 智能分析（AIAnalysis）

AI 漏洞分析与安全顾问对话。

- 两种模式：**漏洞分析** / **AI 安全顾问聊天**。
- 漏洞选择器；风险评分圆环。
- AI 分析详情、修复建议（根因 / 方案 / 代码 / 验证）。
- 聊天可关联漏洞上下文、清空历史。
- **状态检测**：当 AI 服务不可达时自动回退本地规则引擎。
- 查看 / 清空 AI 分析历史。

#### 5.3.3 知识库（KnowledgeBase）

安全知识文章中心，分类包括：web 安全、软件供应链、数据安全、运维流程、工具指南、事件案例、合规、综合。

- 侧边栏：搜索、分类、标签、热门、最新。
- 文章网格；按更新/浏览/发布时间排序。
- 高频 CWE 但缺文章时提示撰写。
- 「写文章」跳转编辑器。

**知识库文章详情 / 编辑器**：Markdown 渲染；可编辑、删除（仅 `admin` / `security_analyst`）；编辑器支持标题、分类、标签、摘要、正文、发布、预览。

### 5.4 配置

#### 5.4.1 项目管理（Projects）

管理被扫描的代码项目。

- 项目卡片网格；新建 / 编辑 / 删除。
- 字段：名称、仓库 URL、目标 URL、本地路径、语言、类型、描述（URL 自动规范化）。
- 分页 / 搜索。

#### 5.4.2 工具集成（Tools）

注册与管理第三方安全工具（SAST / SCA / DAST / SECRET 四类）。

- 工具卡片展示；注册 / 编辑 / 删除。
- 启用 / 禁用切换；连接测试（test）。
- 展开查看关联知识库文章。

#### 5.4.3 资产管理（Assets）

管理应用资产（由项目同步或手动登记），关联开放漏洞与风险分。

- 统计卡片、风险分布条。
- 搜索 + 风险 / 类型 / 状态筛选。
- 表格 / 卡片视图切换。
- 从项目一键同步资产。
- 新增资产；查看关联开放漏洞；重算风险分；删除。

#### 5.4.4 规则管理（Rules）

自定义扫描规则与忽略规则。

- 统计、类型筛选（custom_scan / ignore）、分类筛选（sast/sca/secret/dast/generic）。
- 新建 / 编辑规则：名称、类型、分类、严重度多选、模式表达式、说明、启用、全局/项目级范围。
- 启用/禁用切换；删除；分页。

### 5.5 合规

#### 5.5.1 报告中心（Reports）

生成与下载安全报告。

- **报告类型**：安全总览 / 漏洞明细 / SLA 合规 / 趋势分析 / 合规检查清单。
- **格式**：JSON / Markdown / CSV（另支持 PDF 下载）。
- 生成报告；按类型筛选；详情查看；下载 / 删除。

#### 5.5.2 审计日志（AuditLog）

记录平台所有操作与安全事件，用于合规追溯。

- 统计概览；三个标签：操作日志 / 统计分布 / 安全事件。
- 多维度筛选：操作类型、资源、结果、风险等级、日期区间。
- 自动刷新（15 秒）；导出 CSV；详情弹窗。

### 5.6 系统

#### 5.6.1 用户管理（UserManagement）— 仅 admin

管理平台用户账号与角色。

- 统计卡片；搜索；角色 / 状态筛选；用户表格。
- 角色下拉切换、启用 / 禁用、解锁账号、删除用户。
- **创建用户**：邮箱 / 密码（≥8 位）/ 角色 / 状态（邀请模式入口）。
- 角色权限速览面板。

> 侧边栏「用户管理」仅 `admin` 可见；「创建用户」按钮仅 `admin` 可点击。

#### 5.6.2 系统设置（Settings）

平台通知、CI/CD 门禁、备份与外观配置（6 个标签页）：

- **邮件通知**：SMTP 配置（服务器、端口、账号、密码、发件人、收件人）、按严重级别开关邮件、保存、发送测试邮件。
- **告警规则**：漏洞发现通知、每日安全摘要、每周安全报告开关。
- **门禁策略**：门禁模式（阻断 / 告警）、各级别处置规则、规则预览、Webhook Token（只读 + 重新生成）。
- **数据备份**：生成快照、上传文件恢复（恢复前自动备份）、下载快照、列表展示。
- **通知渠道**：配置飞书机器人 Webhook（名称 / 地址 / Secret / 启用）、渠道列表启用停用删除。
- **外观**：上传 / 移除 Logo（仅 admin）；注册策略开关（仅 admin，开放公开注册）。

> 「Logo 上传/移除」「注册策略开关」仅 `admin` 可操作；其余设置全员可进入查看。

## 6. 安装与部署

### 6.1 环境要求

- Python 3.11+（后端）
- Node.js 18+（前端构建）
- Docker / Docker Compose（推荐部署方式）
- SQLite（内置，无需独立数据库服务）

### 6.2 Docker Compose 部署（推荐）

项目根目录已提供 `docker-compose.yml`、`Dockerfile.backend`、`Dockerfile.frontend`、`nginx.conf`。

```bash
# 进入项目根目录
cd sentinel-security
```

### 6.3 本地分离运行（开发/调试）

**后端**：

```bash
cd sentinel-security/backend
python -m venv venv && source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt

# 必须指定数据库路径（指向真实库，勿指向其他位置的空库）
export SENTINEL_DB_PATH="$(pwd)/sentinel.db"        # Windows 用绝对路径，如 C:/path/to/backend/sentinel.db

```

**前端**：

```bash
cd sentinel-security/frontend
npm install
npm run build        # 生产构建到 dist/，由后端静态托管
# 或 npm run dev     # 开发模式
```

> 后端启动后默认在 5000 端口同时提供 API 与前端 `dist/` 静态资源，直接访问 `http://127.0.0.1:5000` 即可，无需单独启动前端 dev server。

### 6.4 关键配置

- **数据库路径**：通过环境变量 `SENTINEL_DB_PATH` 指定 SQLite 文件路径。**务必指向真实业务库**，错误地指向空库会导致登录失败、扫描无数据。
- **JWT 密钥**：由 `backend/.jwt_secret`（或 `config.JWT_SECRET`）提供，部署时请替换为强随机值。
- **示例配置**：参考项目根目录 `.env.example` 与 `backend/.env.example`。

## 7. 集成与自动化

### 7.1 CI/CD 安全门禁

平台可将安全卡点嵌入研发流水线：

- 在「扫描中心」或「设置 → 门禁策略」中获取 GitLab / GitHub / Jenkins 集成代码与 Webhook Token。
- 配置门禁规则，例如：Critical → 阻断合并；High → 告警；Medium / Low → 放行。
- 重新生成 Webhook Token（旧 Token 立即失效）。

### 7.2 定时调度

在「扫描中心」配置定时扫描：

- 支持 cron 或固定间隔（interval）。
- 可启用/停用、立即执行、编辑、删除。

### 7.3 通知集成

- **邮件通知**：在「设置 → 邮件通知」配置 SMTP，可按漏洞严重级别发送通知，并开启每日/每周安全摘要。
- **飞书机器人**：在「设置 → 通知渠道」配置 Webhook 地址与 Secret，用于告警推送。

### 7.4 数据备份与恢复

在「设置 → 数据备份」中：

- 一键生成数据快照（平台会自动备份当前数据）。
- 下载快照用于离线留存。
- 上传快照执行恢复（恢复前会自动备份，避免覆盖丢失）。
- 此功能与第 9 章的运维备份互补，建议定期快照。

## 9. 常见问题与运维提示

### 9.1 登录提示「邮箱或密码错误」

- 确认账号密码正确；默认管理员密码为 `admin123`（首次登录需改密）。
- 若账号被禁用，请联系 `admin` 在「用户管理」中启用/解锁。
- **数据库路径错误**是常见根因：确保后端启动时 `SENTINEL_DB_PATH` 指向**真实业务库**，而非其他位置的空库（空库无用户记录，会报密码错误）。

### 9.2 每次登录都要求改密码

强制改密仅在「密码仍为默认 `admin123`」时触发，改密成功后不再弹出。若反复弹出，多为旧浏览器缓存导致——请**硬刷新（Ctrl+F5）**加载最新前端，并清除 localStorage 中残留的 `sentinel_force_pwd` 标记。

### 9.3 扫描无法发起 / 删除按钮无效

- 确认账号具有相应权限（发起扫描需 `admin` 或 `security_analyst`；删除扫描/漏洞/报告等仅 `admin`）。
- 确认后端服务正常（健康检查：`GET /api/auth/register/status` 应返回 200）。
- 检查数据库路径是否正确指向真实库。

### 9.4 数据库运维建议

- 平台使用 SQLite。多进程/被强杀场景下建议使用 **DELETE 日志模式**（而非 WAL），避免残留 `-wal`/`-shm` 锁文件导致启动失败。
- 定期通过「设置 → 数据备份」生成快照，或拷贝 `backend/sentinel.db` 文件进行离线备份。
- **恢复时务必确认目标为真实业务库**，避免覆盖正确数据。

### 9.5 注册入口不显示

平台默认关闭公开注册（邀请模式）。登录页仅当管理员在「设置 → 外观 → 注册策略」开启后才显示注册入口；否则需由管理员在「用户管理」创建账号。

### 9.6 前端样式异常

前端依赖 Tailwind。若构建时遗漏 `tailwind.config.cjs` / `postcss.config.cjs`，会导致样式丢失。请使用项目自带构建配置重新 `npm run build`。

## 10. 安全与合规说明

- **认证安全**：JWT（HS256）+ `token_version` 吊销机制，密码采用 PBKDF2-HMAC-SHA256（26 万轮，带随机盐）；改密后旧 Token 立即失效。
- **访问控制**：基于角色的 RBAC，敏感写/删操作仅 `admin` 可执行；界面按钮可见性不代表权限，以服务端校验为准。
- **默认安全**：默认关闭公开注册；默认管理员首次登录强制改密；外部接口对路径穿越、SQL 注入、命令注入均有防护。
- **审计追溯**：全量操作记录与安全事件日志，支持检索与导出，满足内部审计与合规要求。
- **数据安全**：建议定期备份数据库与平台快照；备份文件请妥善保管，避免泄露业务数据。
